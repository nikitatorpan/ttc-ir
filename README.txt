TealTech LP Portal Demo / Портал для инвесторов

Эта папка содержит статический портал LP для фонда «ТилТех Капитал». По умолчанию данные берутся из файла `data/config.js`, но вы можете подключить таблицу Google, чтобы редактировать контент через удобный интерфейс.

### Содержимое

- **index.html** — главная страница. Подключает `styles.css` и `scripts.js`.
- **styles.css** — стили, соответствующие брендбуку (цвета, размеры, шрифты). Здесь подключаются шрифты Oswald и Montserrat.
- **logo.svg** — логотип, который используется в шапке сайта.
- **data/config.js** — массив демонстрационных записей (новости, отчёты, документы, портфель компаний). Можно заменить или расширить.
- **scripts.js** — скрипт, который отображает элементы, фильтрует по категориям и поиску, а также может загружать данные из Google Sheets.

### Подключение Google Sheets для редактирования

Портал может загружать данные непосредственно из Google Sheets через официальный API. Это позволяет обновлять содержание сайта без правок кода — достаточно отредактировать таблицу.

1. **Подготовьте таблицу.** Используйте структуру, аналогичную примеру `IR для сайта`:
   - Столбец A: порядковый номер (может быть пустым).
   - Столбец B: **Название документа** (название компании или файла).
   - Столбец C: **Категория**. Допустимые значения на русском: «Отчёты», «Новости», «Документы», «Портфель». Можно добавлять и другие категории — они будут автоматически подхвачены (будет использоваться её оригинальное значение).
   - Столбец D: **Дата обновления** (например, `31.07.2025`).
   - Столбец E: **Пояснение** (краткое описание).
   - Столбец F: **Ссылка** (URL на документ или папку в облаке).
   Первая строка должна содержать заголовки (как в примере) — они будут пропущены при загрузке.

2. **Настройте доступ.** Таблица должна быть доступна для чтения: нажмите «Share» → «Anyone with the link» → «Viewer». Это позволит API читать данные.

3. **Получите API‑ключ.**
   - Перейдите в [Google Cloud Console](https://console.cloud.google.com/) и создайте новый проект.
   - Включите API **Google Sheets** для этого проекта.
   - В разделе «Credentials» создайте API‑ключ. Ограничьте его доменом вашего сайта (реферером) для безопасности.

4. **Укажите идентификаторы в `scripts.js`.** Откройте файл `scripts.js` и в функции `loadFromSheet()` задайте:
   - `SHEET_ID` — идентификатор вашей таблицы (часть URL между `/d/` и `/edit`).
   - `API_KEY` — ваш API‑ключ Google.
   - `RANGE` — диапазон, который нужно загружать. По умолчанию `Лист1!A:F` загружает шесть первых столбцов с листа `Лист1`. Если ваш лист назван иначе, измените название перед `!`.

5. **Загрузите файлы на хостинг.** После сохранения изменений загрузите файлы на сервер. При открытии страницы сайт вызовет Google Sheets API, получит свежие данные и отобразит их.

Если вы не укажете `SHEET_ID` или `API_KEY`, портал будет использовать встроенный массив `window.portalItems` из `data/config.js`. Это позволяет разместить несколько статических записей без подключения к таблице.

### Развёртывание

Чтобы развернуть портал:

1. Сожмите содержимое папки `lp_portal_demo` в ZIP-архив (можно использовать готовый архив `tealtech_lp_portal_ru_updated.zip`).
2. Загрузите архив на любой статический хостинг (GitHub Pages, Netlify, Vercel, Tiiny.host и т. п.). Важно, чтобы хостинг обслуживал `index.html` как главную страницу.
3. Если хотите использовать парольную защиту, убедитесь, что пароль указан в `index.html` (по умолчанию — «Blacktech»). Для более надёжной защиты рекомендуем установить Basic Auth на уровне сервера.

### Расширение списка категорий

Вы можете добавлять новые категории в таблицу Google или в массив `portalItems`. Чтобы вывести русское название категории на карточке, добавьте соответствующую запись в объект `categoryNames` в файле `scripts.js`.

### Поддержка и доработка

Этот портал разработан как простая платформа для размещения документов. Вы можете изменять стили, структуру и логику, не затрагивая основной принцип: все данные хранятся в одном месте и загружаются на сайт автоматически.
